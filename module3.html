Real Examples: How We Used AI

Actual Scenarios from Our Project

Example 1: Debugging OpenWrt Configuration

Problem: Our VPN wasn't routing traffic correctly. Spent 30 mins googling with no luck.

My prompt to Claude: "I'm setting up WireGuard VPN on OpenWrt. Traffic is not routing through the VPN tunnel even though the connection shows as established. Here's my config file: [pasted config]. What am I missing?"
Claude's response: Identified that we were missing the 'AllowedIPs' configuration and firewall rules weren't forwarding traffic correctly. Provided exact commands to fix it.

Result: Fixed in 15 minutes. Learned WHY it was broken, not just how to fix it.

Example 2: Generating Web Interface Code

Need: Create a dashboard to display connected devices in real-time.

My prompt to GitHub Copilot (as a comment in code): // Function to fetch connected devices from router API and display in table
// Copilot generated: async function fetchConnectedDevices() { try { const response = await fetch('/api/devices'); const devices = await response.json(); const tableBody = document.getElementById('devices-table'); tableBody.innerHTML = ''; devices.forEach(device => { const row = ` ${device.hostname} ${device.ip} ${device.mac} ${device.connected_time} `; tableBody.innerHTML += row; }); } catch (error) { console.error('Error fetching devices:', error); } }

Result: Got working code in 30 seconds. We then customized it to add filtering and sorting.

The Art of Prompting

Good Prompts vs. Bad Prompts

‚ùå Bad Prompt

"How do I make a VPN?"

Too vague - AI doesn't know what you need

‚úÖ Good Prompt

"I'm setting up WireGuard VPN on a Raspberry Pi running OpenWrt. I need to configure it so all traffic from connected clients routes through the VPN. Can you explain the key configuration parameters and provide an example config?"

Specific context + clear goal

‚ùå Bad Prompt

"Fix my code"

No code, no context, no error

‚úÖ Good Prompt

"This JavaScript function is supposed to fetch device data from my API but I'm getting a CORS error. Here's the code: [paste code]. Here's the error: [paste error]. What's wrong and how do I fix it?"

Includes code + error + expected behavior

‚ùå Bad Prompt

"Write my senior project"

Academic integrity violation!

‚úÖ Good Prompt

"Can you explain the difference between WireGuard and OpenVPN in terms of performance and security? I'm writing the literature review section of my report and want to justify why we chose WireGuard."

Uses AI to learn and support your own work

üí° Formula for Great Prompts:

Context + Specific Question + Expected Outcome

Example: "I'm building [context]. I'm trying to [goal] but [problem]. [Specific question]?"

AI as Your Personal Tutor

Learn Faster, Understand Deeper

‚úÖ Using AI to LEARN (Good!)

Scenario: You don't understand firewall chains

"I'm new to iptables and firewall configuration. Can you explain what INPUT, OUTPUT, and FORWARD chains do, using a simple analogy? Then show me a basic example for a router."

Why this works: You're asking AI to TEACH you, not DO for you. You'll understand the concept and can apply it yourself.

Scenario: Code isn't working, but you want to understand why

"This function should convert IP addresses to hostnames but it's not working. Can you walk me through what each line does and identify where the logic breaks down? [paste code]"

Why this works: You're debugging WITH AI, learning the process, not just getting a fixed version.

‚ùå Using AI to AVOID Learning (Bad!)

  • "Write me a complete firewall configuration" ‚Üí You paste it without understanding, breaks later, you're stuck
  • "Debug this for me" ‚Üí AI fixes it, you don't learn how to debug next time
  • "Explain X" ‚Üí You read the explanation once, forget it, ask again next week

Better approach: "Explain X, then quiz me on it" or "Explain X and give me a practice problem"

üí° The Iterative Learning Pattern:

  1. Ask AI to explain concept simply
  2. Try to implement it yourself
  3. When stuck, ask AI specific questions about where you're stuck
  4. Ask AI to review your solution and suggest improvements
  5. Repeat until you truly understand

"I'm writing the Methodology section of my senior project report about a smart router. I've implemented these features: [list]. Can you suggest a logical structure for this section that flows well?"

Use Case 2: Improving Technical Writing

AI helps with: Tone, clarity, technical vocabulary

Use Case 3: Generating Diagram Descriptions

"I have a network topology diagram showing: [describe diagram]. Help me write a clear caption and a paragraph explaining what the diagram shows for my report."

AI helps with: Clear explanations of visual content

‚ö†Ô∏è What NOT to do:

  • Don't copy-paste AI-generated text without understanding it

Academic Integrity: The Rules

What's Okay vs. What's Not

‚úÖ ACCEPTABLE Use of AI:

  • Learning & Understanding: "Explain how X works"
  • Debugging: "Why is my code throwing this error?"
  • Code Review: "Review my code for security issues"
  • Boilerplate: "Generate a basic REST API structure" (then customize)
  • Improving Writing: "Make this paragraph clearer"
  • Brainstorming: "What are potential solutions to [problem]?"
  • Research Help: "Summarize this concept from these papers"

Key principle: AI assists YOUR work, doesn't replace YOUR thinking.

‚ùå UNACCEPTABLE Use of AI:

  • "Write my entire project"
  • "Generate complete report sections" (without understanding/editing)
  • "Do my homework/assignments"
  • Copying AI code without understanding it
  • Submitting AI-written content as your own words
  • Using AI to fabricate data or results

These violate academic integrity and could get you in serious trouble.

üí° How to Acknowledge AI Use:

In your report's acknowledgments or methodology:

"AI tools (Claude, GitHub Copilot) were used to assist with debugging code, understanding technical concepts, and improving report clarity. All final implementations and written content are original work by the authors."

Check your university's specific policies! LIU may have guidelines - ask your advisor.

‚ö†Ô∏è The "Understanding Test"

Before using AI-generated code or text, ask yourself:

  • Can I explain how this works to my advisor?
  • Could I modify this if requirements changed?
  • Do I understand why this is the right approach?

If you answer "no" to any of these, you need to learn more before using it!

Prompt Templates for Your Project

Copy & Customize These

Template 1: Understanding Concepts

"I'm working on [project topic]. Can you explain [concept] in the context of [specific technology]? Use analogies if helpful, and provide a simple example."

Example: "I'm working on a smart router. Can you explain Quality of Service (QoS) in the context of home networking? Use analogies if helpful."

Template 2: Debugging

"I'm implementing [feature] using [technology]. I'm getting this error: [error message]. Here's my code: [paste code]. What's causing this error and how can I fix it?"

Example: "I'm implementing device monitoring using JavaScript fetch API. I'm getting CORS error. Here's my code: [code]. What's causing this?"

Template 3: Code Review

"Please review this [type of code] for [specific concerns like security, performance, best practices]. Here's the code: [paste code]. Suggest improvements."

Example: "Please review this firewall configuration script for security vulnerabilities. Here's the code: [code]."

Template 4: Report Writing Help

"I'm writing the [section name] of my technical report. Here's my rough draft: [paste text]. Can you suggest improvements for clarity and technical accuracy? Keep the main ideas but make it more professional."

Example: "I'm writing the Implementation section. Here's my rough draft: [text]. Suggest improvements for clarity."

Template 5: Comparing Technologies

"I'm deciding between [Technology A] and [Technology B] for [specific use case]. Can you compare them based on [criteria like performance, ease of use, community support]? I need this for my literature review."

Example: "I'm deciding between WireGuard and OpenVPN for my router VPN. Compare them based on speed and ease of configuration."

Common AI Mistakes to Avoid

Learn from Others' Errors

‚ùå Mistake #1: Blindly Trusting AI

Scenario: Student asks AI for firewall rules, copies them without understanding.

Result: Rules block legitimate traffic. Student can't debug because they don't understand the rules.

Lesson: ALWAYS verify AI suggestions. Test them. Understand them.

‚ùå Mistake #2: Not Providing Enough Context

Bad prompt: "Fix this code" [pastes 500 lines]

Better: "This function at line 47 should [expected behavior] but instead [actual behavior]. Here's the relevant code: [paste 20 lines]"

Lesson: Isolate the problem. Give context. Be specific.

‚ùå Mistake #3: Using AI as a Crutch

Scenario: Student asks AI for help with EVERY single line of code. Never tries to solve anything themselves first.

Result: Doesn't learn. Can't answer basic questions during presentation.

Lesson: Try for 15-30 min yourself FIRST. Then ask AI for help if stuck.

‚ùå Mistake #4: Ignoring AI's Limitations

AI can:

  • ‚ùå Be confidently wrong about obscure technical details
  • ‚ùå Generate code that looks right but has subtle bugs
  • ‚ùå Miss context-specific requirements
  • ‚ùå Produce outdated information (especially for new tech)

Lesson: Cross-reference important info. Test all code. Use your judgment.

‚ùå Mistake #5: Not Iterating on Prompts

Scenario: First AI response isn't helpful. Student gives up.

Better approach: Refine the prompt. Add more context. Try different phrasing. Have a conversation.

Lesson: Prompting is a skill. It takes iteration.
