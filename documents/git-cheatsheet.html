Essential Commands for Senior Projects

1. Initial Setup

Configure Git for the First Time

git config --global user.name "Your Name"
Set your name for all commits
git config --global user.email "your.email@example.com"
Set your email for all commits

Start a New Repository

git init
Initialize a new Git repository in current folder
git clone https://github.com/username/repo.git
Copy an existing repository to your computer

2. Daily Workflow

The Golden Rule: Commit early, commit often. Small commits are your friend!

Check Status

git status
See which files have changed
git diff
See exactly what changed in your files

Stage and Commit

git add filename.txt
Stage specific file for commit
git add .
Stage ALL changed files for commit
git commit -m "Added VPN configuration module"
Save your changes with a message

‚úÖ Good commit messages:

  • "Added authentication system"
  • "Fixed bug in database connection"
  • "Updated documentation for API endpoints"

Push to GitHub

git push
Upload your commits to GitHub
git push origin main
Push to specific branch (main or master)

3. Working with GitHub

Connect Your Local Repo to GitHub

git remote add origin https://github.com/username/repo.git
Link your local repo to GitHub
git remote -v
View all remote connections

First Time Push to GitHub

git push -u origin main
Push to GitHub and set main as default branch

Getting Updates

git pull
Download partner's changes from GitHub
git fetch
Check for updates without merging yet

4. Branching Workflow

Why branches? Keep main stable. Develop features on separate branches. Merge when ready.

Create and Switch Branches

git branch feature/vpn-setup
Create new branch
git checkout feature/vpn-setup
Switch to that branch
git checkout -b feature/vpn-setup
Create AND switch to new branch (shortcut)

List Branches

git branch
List all local branches
git branch -a
List all branches (local + remote)

Merge Branches

git checkout main
Switch back to main branch
git merge feature/vpn-setup
Merge feature branch into main

Delete Branches

git branch -d feature/vpn-setup
Delete merged branch
git push origin --delete feature/vpn-setup
Delete branch from GitHub

5. Collaborating with Partners

Before Each Work Session

git pull
Get latest changes from partner

During Work Session

git status
Check what you've changed
git add .
Stage your changes
git commit -m "Implemented device monitoring"
Save with clear message

After Work Session

git push
Upload to GitHub for partner to access
‚ö†Ô∏è Conflict Prevention:
  • Pull before you start working
  • Work on different files/branches when possible
  • Commit and push frequently
  • Communicate: "I'm working on X, don't touch it"

If Conflict Happens

git status
See which files have conflicts
git merge --abort
Cancel merge and start over

6. Fixing Mistakes

Before You Run These Commands: Make sure you understand what you're undoing!

Undo Unstaged Changes

git checkout -- filename.txt
Discard changes to a file (not yet staged)
git restore filename.txt
Modern way to discard changes

Undo Staged Changes

git reset HEAD filename.txt
Unstage a file

Undo Commits

git revert HEAD
Undo last commit (creates new commit)
git reset --soft HEAD~1
Undo last commit, keep changes staged
git reset --hard HEAD~1
‚ö†Ô∏è DANGER: Completely remove last commit

View History

git log
See all your commits
git log --oneline
See commits in compact format

8. Git Best Practices for Senior Projects

‚úÖ DO:

  • Commit frequently (multiple times per day)
  • Write clear, descriptive commit messages
  • Pull before you start working
  • Push at end of each work session
  • Use branches for new features
  • Test locally before pushing
  • Communicate with partner about what you're working on

‚ùå DON'T:

  • Commit everything at the last minute
  • Write vague messages like "fixed stuff"
  • Force push without checking first
  • Work on main branch (use feature branches)
  • Leave merge conflicts unresolved
  • Commit binary files or node_modules
