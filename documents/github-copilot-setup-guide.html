1. What is GitHub Copilot?

GitHub Copilot is an AI pair programmer that sits inside your code editor. It suggests code as you type ‚Äî from single lines to entire functions ‚Äî based on context from your code and comments.

üöÄ What It Can Do

  • Autocomplete code as you type
  • Generate functions from comments
  • Suggest multiple code alternatives
  • Write boilerplate & repetitive code
  • Help with unfamiliar languages/APIs
  • Generate unit tests

üéì Why It's Great for Senior Projects

  • Saves hours on boilerplate code
  • Helps learn new frameworks faster
  • Reduces syntax errors
  • Speeds up prototyping
  • Works with any programming language
  • FREE for students!

üí° Languages Supported: Python, JavaScript, TypeScript, C, C++, C#, Java, Go, Ruby, PHP, Shell scripting, HTML/CSS, SQL, and many more. It works best with popular languages that have lots of training data.

3. Set Up Copilot in VS Code

VS Code is the recommended editor for most senior projects. Here's how to install Copilot:

1

Install VS Code

Download from code.visualstudio.com if you haven't already.

2

Install the GitHub Copilot Extension

  • Open VS Code
  • Press Ctrl+Shift+X to open Extensions
  • Search for "GitHub Copilot"
  • Click Install on the one by GitHub (verified publisher)
  • Also install "GitHub Copilot Chat" for the chat interface
3

Sign In to GitHub

  • After installing, you'll see a prompt to sign in
  • Click "Sign in to GitHub"
  • A browser window opens ‚Äî log in with your GitHub account
  • Authorize VS Code to access your account
  • Return to VS Code ‚Äî you should see the Copilot icon in the status bar
4

Verify It's Working

Create a new file (e.g., test.py) and type:

# Function to calculate the factorial of a number

Press Enter and wait a moment. Copilot should suggest the complete function in gray text. Press Tab to accept.

‚úÖ If you see gray suggestion text: Copilot is working!

‚ö†Ô∏è If nothing appears: Check the Copilot icon in the bottom-right status bar. It should show as active (not crossed out).

5. Your First Time Using Copilot

Here's how Copilot works in practice:

üìù Inline Suggestions (Ghost Text)

As you type, Copilot shows suggestions in gray "ghost text":

# Type this comment: # Function to read a CSV file and return a list of dictionaries # Copilot suggests the complete function below: def read_csv(filepath): import csv with open(filepath, 'r') as file: reader = csv.DictReader(file) return list(reader)

üí¨ Comment-Driven Development

Write descriptive comments and Copilot generates the code:

// JavaScript example: // Fetch all connected devices from the router API, // filter out inactive ones, and sort by connection time async function getActiveDevices() { const response = await fetch('/api/devices'); const devices = await response.json(); return devices .filter(device => device.status === 'active') .sort((a, b) => new Date(b.connected_at) - new Date(a.connected_at)); }

üîÑ Multiple Suggestions

Copilot can offer different alternatives for the same code:

  • Tab ‚Äî Accept the current suggestion
  • Esc ‚Äî Dismiss the suggestion
  • Alt+] ‚Äî See next alternative suggestion
  • Alt+[ ‚Äî See previous alternative suggestion
  • Ctrl+Enter ‚Äî Open Copilot completions panel (see up to 10 suggestions)

7. Using Copilot Chat

Copilot Chat is a built-in AI assistant inside VS Code. Think of it as having Claude or ChatGPT right in your editor.

How to Open Copilot Chat

  • Chat Panel: Press Ctrl+Shift+I (opens sidebar chat)
  • Inline Chat: Press Ctrl+I (opens chat right in your code)
  • Quick Chat: Press Ctrl+Shift+Alt+L

Useful Chat Commands

/explain ‚Üí Explain selected code /fix ‚Üí Fix problems in selected code /tests ‚Üí Generate unit tests for selected code /doc ‚Üí Generate documentation/comments @workspace ‚Üí Ask about your entire project #file ‚Üí Reference a specific file in your question

Example Chat Prompts for Senior Projects

Debugging: "Why is this function returning null? I expect it to return a list of devices."

Explaining code: Select a block of code ‚Üí Ctrl+I ‚Üí "Explain what this does step by step"

Refactoring: Select code ‚Üí Ctrl+I ‚Üí "Refactor this to use async/await instead of callbacks"

Testing: Select a function ‚Üí Ctrl+I ‚Üí "/tests"

8. Best Practices for Senior Projects

‚úÖ DO These Things

  • Write clear comments first ‚Äî Copilot generates better code from good comments
  • Review every suggestion ‚Äî Don't Tab-accept blindly
  • Use descriptive variable names ‚Äî Helps Copilot understand context
  • Break problems into small functions ‚Äî Copilot handles small, focused functions best
  • Use Copilot Chat to understand generated code ‚Äî Select it and ask "/explain"
  • Check for security issues ‚Äî AI-generated code isn't always secure
  • Test all generated code ‚Äî It might look right but have subtle bugs

‚ö†Ô∏è DON'T Do These Things

  • Don't accept suggestions without reading them
  • Don't paste sensitive data (passwords, API keys) where Copilot can see them
  • Don't rely on Copilot for complex algorithms without verification
  • Don't let it write your entire project ‚Äî you need to understand every line
  • Don't skip testing because "the AI wrote it"

üí° Pro Tips from Our Project

  • Name your files well: device_monitor.py gives Copilot better context than script1.py
  • Keep related files open: Copilot uses open tabs as context
  • Write function signatures first: Let Copilot fill in the body
  • Use it for boilerplate: Config files, API endpoints, HTML templates ‚Äî perfect for Copilot
  • Pair with Copilot Chat: When inline suggestions aren't enough, chat for complex problems

9. Troubleshooting Common Issues

‚ùå "Copilot is not active" / No suggestions appearing

  • Check the Copilot icon in the VS Code status bar (bottom right)
  • If it's crossed out, click it and enable it
  • Make sure you're signed into GitHub in VS Code
  • Try: Ctrl+Shift+P ‚Üí "GitHub Copilot: Sign In"
  • Restart VS Code

‚ùå "Your GitHub account doesn't have Copilot access"

  • Verify your Student Developer Pack is approved at education.github.com/pack
  • Go to github.com/settings/copilot and enable it
  • If your pack isn't approved yet, wait for the email confirmation

‚ùå Suggestions are irrelevant / low quality

  • Add more context: better comments, clearer variable names
  • Open related files in VS Code tabs (Copilot uses them as context)
  • Try pressing Alt+] to cycle through alternative suggestions
  • Use Copilot Chat instead for complex requests

‚ùå Copilot is slow / laggy

  • Check your internet connection (Copilot needs internet)
  • Close unnecessary VS Code extensions
  • Check GitHub's status page: githubstatus.com
  • Restart VS Code
